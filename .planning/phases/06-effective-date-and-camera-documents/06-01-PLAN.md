# Plan 06-01: Store + Validation + OCR

## Goal
Add `effectiveDate` field to store, validation schema, OCR schema, and OCR prompt for automatic extraction from business permit.

## Files to modify
- `src/store/useApplicationStore.ts`
- `src/lib/validation.ts`
- `src/lib/ocr-schema.ts`
- `src/app/api/extract-permit/route.ts`

## Steps

### 1. Update useApplicationStore.ts
- Add `effectiveDate: string` to `BusinessInfoData` interface (line ~17)
- Add `effectiveDate: ''` to `initialBusinessInfo` constant (line ~94)

### 2. Update validation.ts
- Add to `businessInfoSchema`:
  ```typescript
  effectiveDate: z
    .string()
    .min(1, 'Effective date is required')
    .refine((val) => {
      const date = new Date(val);
      const today = new Date();
      today.setHours(0, 0, 0, 0);
      return date >= today;
    }, 'Effective date cannot be in the past')
    .refine((val) => {
      const date = new Date(val);
      const maxDate = new Date();
      maxDate.setDate(maxDate.getDate() + 30);
      return date <= maxDate;
    }, 'Effective date cannot be more than 30 days out'),
  ```

### 3. Update ocr-schema.ts
- Add to `businessPermitOcrSchema`:
  ```typescript
  effectiveDate: z
    .string()
    .describe('Permit issue date or effective date in YYYY-MM-DD format')
    .optional()
    .or(z.literal('')),
  ```

### 4. Update extract-permit/route.ts
- Add to textPrompt extraction instructions:
  ```
  - Effective Date / Permit Issue Date - the date the permit was issued (usually in YYYY-MM-DD format or readable as a date)
  ```
- Ensure the parsed OCR data is returned correctly (should auto-map since field names match)

## Verification
- Type check passes: `npm run typecheck` or `npm run lint`
- No runtime errors when form renders
- OCR extracts date when present in permit image
