# Plan 06-02: Effective Date UI

## Goal
Add effective date date picker input to the ASSURED DETAILS section in BusinessInfoStep.tsx.

## Files to modify
- `src/app/apply/steps/BusinessInfoStep.tsx`

## Steps

### 1. Register effectiveDate in form
- Add `effectiveDate: businessInfo.effectiveDate` to `defaultValues` in the `useForm` hook (around line 98)
- Note: effectiveDate is optional in the schema (can be empty for manual entry without OCR)

### 2. Add date input in ASSURED DETAILS section
- Find the `grid grid-cols-2` section containing `tin` and `floorArea` (line 302-325)
- Replace or modify to add `effectiveDate` after `floorArea` in the 2-col grid, or add as a new row below
- Use the existing pattern:
  ```tsx
  <div className="group">
    <UnderlineLabel htmlFor="effectiveDate">Effective Date</UnderlineLabel>
    <input
      id="effectiveDate"
      type="date"
      className="input-underline"
      {...register('effectiveDate')}
    />
    <FieldError message={errors.effectiveDate?.message} />
  </div>
  ```

### 3. Set min/max constraints on the date input
- Add `min` and `max` attributes programmatically:
  - min = today's date in YYYY-MM-DD format
  - max = 30 days from now in YYYY-MM-DD format
- Use a useMemo or useEffect to calculate these values

### 4. Set default value
- In the form's `defaultValues`, set `effectiveDate` to today's date (YYYY-MM-DD)
- This ensures the field has a sensible default when user opens the form

### 5. Pre-fill from OCR scan data (optional, in useEffect)
- If `scanData.effectiveDate` exists, call `setValue('effectiveDate', scanData.effectiveDate)`
- This should work automatically since `effectiveDate` is part of `BusinessInfoFormData`

## Verification
- Date picker appears in ASSURED DETAILS section
- Default shows today's date
- Calendar restricts selection to today through +30 days
- No console errors
- Form submits successfully with the date value

## Boundaries
- Do NOT change any other fields in the form
- Do NOT modify validation logic beyond date constraints
- Keep underline input styling consistent
- Maintain onBlur validation mode
