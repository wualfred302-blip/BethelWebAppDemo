  <name>Task 2: Create Zod validation schemas</name>
  <files>src/lib/validation.ts</files>
  <action>
    Create `src/lib/validation.ts` with Zod schemas for all three steps.

    `businessInfoSchema`:
    - `natureOfBusiness`: string, min 1, error "Please select a nature of business"
    - `businessName`: string, min 2 max 200, error "Business name is required (2-200 characters)"
    - `tin`: string, regex `^\d{3}-\d{3}-\d{3}-\d{3}$`, error "TIN must be in format 123-456-789-000"
    - `floorArea`: string, regex `^\d+(\.\d{1,2})?$`, error "Floor area must be a valid number"

    `locationSchema`:
    - `regionCode`: string, min 1, error "Please select a region"
    - `regionName`: string, min 1
    - `provinceCode`: string, min 1, error "Please select a province"
    - `provinceName`: string, min 1
    - `cityCode`: string, min 1, error "Please select a city/municipality"
    - `cityName`: string, min 1
    - `barangayCode`: string, min 1, error "Please select a barangay"
    - `barangayName`: string, min 1

    `contactSchema`:
    - `phone`: string, regex `^(\+63|09)\d{9}$`, error "Enter a valid PH mobile number (e.g., 09123456789)"
    - `email`: string, email validation, error "Enter a valid email address"

    Export inferred types: `BusinessInfoFormData`, `LocationFormData`, `ContactFormData`.
  </action>
  <verify>

  </done>

    Each filter function returns `[]` if parent is empty. Use `Object.entries()` to iterate the nested structure. Deduplicate by name. Sort alphabetically.

    Reference the existing `lgu-data.ts` pattern from the BarangayLinkodWebRegistration project for the nested-to-flat transformation approach, but adapt to work directly with the nested JSON (no flat array transformation needed for cascading selects).
