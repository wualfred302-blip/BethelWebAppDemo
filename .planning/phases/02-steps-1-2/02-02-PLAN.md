  <action>
    **Part A — Store update (`useApplicationStore.ts`):**
    Change `DocumentsData` interface from `string | null` to `File | null` for all three fields (businessPermit, dtiSec, validId). Update `initialDocuments` to use `null` (unchanged). The `setDocuments` merge-style setter needs no code change since it uses `Partial<DocumentsData>`.

    **Part B — DocumentsStep (`DocumentsStep.tsx`):**
    Replace the shell component with a full document upload form. Keep the existing `StepProps` interface and Back/Continue button pattern.

    Create 3 document upload slots:
    1. **Business Permit** (required) — label "Business Permit *"
    2. **DTI/SEC Registration** (required — user picks one) — label "DTI/SEC Registration *"
    3. **Valid Government ID** (required) — label "Valid Government ID *"

    Each slot uses an inline UploadZone pattern with:
    - **Drag-and-drop zone**: dashed border area with icon + "Drop file here or click to upload" text
      - `onDragOver`: preventDefault, add highlight class
      - `onDragLeave`: remove highlight class
      - `onDrop`: preventDefault, process file
      - `onClick`: trigger hidden `<input type="file">` ref
    - **File input**: hidden `<input type="file" accept=".jpg,.jpeg,.png,.pdf">`, onChange processes file
    - **Validation on file select**:
      - Check MIME type: reject if not `image/jpeg`, `image/png`, or `application/pdf`. Show "Only JPG, PNG, PDF allowed" error.
      - Check size: reject if > 5MB (5 * 1024 * 1024). Show "File must be under 5MB" error.
    - **After successful upload**:
      - For JPG/PNG: show image preview using `URL.createObjectURL(file)` in a small thumbnail (max 120px)
      - For PDF: show PDF icon + filename + file size
      - Show remove button (X icon or "Remove" text link)
    - **Remove**: clears the file from local state and calls `setDocuments({ [fieldKey]: null })`
    - **State**: local useState per slot for the selected File, plus validation error string
    - **On form submit**: store selected files via `setDocuments({ businessPermit, dtiSec, validId })`, then call `onNext()`

    Layout: max-w-md mx-auto space-y-6. Each upload slot is a div with space-y-2 (label, drop zone, error).

    **Do NOT:**
    - Modify the Select or Input components
    - Add a separate UploadZone.tsx file (inline the pattern per slot)
    - Convert files to base64 (use File objects directly per store change)
    - Add toast notifications for errors (inline red text below the drop zone)
  </action>

    - Uploading a JPG/PNG shows image preview
    - Uploading a PDF shows filename
    - Uploading an oversized file shows "File must be under 5MB" error
    - Uploading a wrong type shows "Only JPG, PNG, PDF allowed" error
    - Remove button clears the upload
