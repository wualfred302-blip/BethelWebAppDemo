  <action>
    Rewrite `src/app/apply/page.tsx` to use exactly 4 steps with browser history integration.

    Changes required:

    1. **Update imports:**
       - Remove imports for LocationStep, DocumentsStep, CoverNoteStep, PaymentStep, SuccessStep
       - Add imports for ReviewPayStep and DoneStep
       - Keep: BusinessInfoStep, ContactCoverageStep, ProgressBar, Button, useApplicationStore, TOTAL_STEPS, motion/AnimatePresence

    2. **Update STEPS array** to exactly 4 entries:
       ```
       const STEPS = [
         { label: 'Business', component: BusinessInfoStep },
         { label: 'Contact', component: ContactCoverageStep },
         { label: 'Review & Pay', component: ReviewPayStep },
         { label: 'Done', component: DoneStep },
       ] as const;
       ```

    3. **Remove Start Over button:** Delete the `reset` usage and the entire "Start Over" `<div>` block (lines 76-82 in current file). Remove `reset` from the destructured store values.

    4. **Update transition animation** from spring to ease-out:
       Replace the current spring transition:
       ```
       transition={{
         x: { type: 'spring', stiffness: 300, damping: 30 },
         opacity: { duration: 0.2 },
       }}
       ```
       With ease-out:
       ```
       transition={{
         x: { duration: 0.2, ease: 'easeOut' },
         opacity: { duration: 0.2 },
       }}
       ```

    5. **Add browser back button support (history state):**
       - Import `useEffect` from React
       - On mount and when `currentStep` changes, push a new history entry:
         ```ts
         useEffect(() => {
           window.history.pushState({ step: currentStep }, '', '/apply');
         }, [currentStep]);
         ```
       - Add a `popstate` event listener to handle browser back:
         ```ts
         useEffect(() => {
           const handlePopState = (e: PopStateEvent) => {
             const step = e.state?.step;
             if (step && step >= 1 && step <= TOTAL_STEPS) {
               goToStep(step);
             }
           };
           window.addEventListener('popstate', handlePopState);
           return () => window.removeEventListener('popstate', handlePopState);
         }, [goToStep]);
         ```
       - Add `goToStep` to the destructured store values

    6. **Add Enter key form submission:**
       - The existing form components (BusinessInfoStep, ContactCoverageStep) already have `<form>` elements with `onSubmit`, so Enter key works natively within those forms.
       - For the placeholder steps (ReviewPayStep, DoneStep), Enter key should trigger `handleNext`.
       - Add a `useEffect` that listens for Enter keypress and calls `handleNext`:
         ```ts
         useEffect(() => {
           const handleKeyDown = (e: KeyboardEvent) => {
             if (e.key === 'Enter' && !e.shiftKey) {
               const target = e.target as HTMLElement;
               // Don't interfere with form submissions (forms handle their own Enter)
               if (target.tagName === 'FORM' || target.closest('form')) return;
               e.preventDefault();
               handleNext();
             }
           };
           window.addEventListener('keydown', handleKeyDown);
           return () => window.removeEventListener('keydown', handleKeyDown);
         }, [handleNext]);
         ```
       - This ensures Enter works on placeholder steps without double-firing on form steps.

    7. **Preserve existing structure:**
       - Keep the white header strip with ProgressBar
       - Keep `max-w-md` centered layout
       - Keep `min-h-screen bg-zinc-50 flex flex-col` root
       - Keep AnimatePresence with `mode="wait"` and `custom={direction}`

    Important: Do NOT add step labels under the dots. The ProgressBar has no labels — this is locked in CONTEXT.md.
